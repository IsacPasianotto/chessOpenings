È possibile utilizzare il programma anche con un proprio dataset. 

Affinché; questo funzioni correttamente, il file in input deve avere estensione .csv, inoltre deve contenere le seguenti variabili così nominate: 

  • winner:   vincitore della partita
  • white_rating:  punteggio ELO del giocatore con i pezzi bianchi
  • black_rating:   punteggio ELO del giocatore con i pezzi neri
  • opening_code:  codice ECO che identifica la partita giocata
  • opening_name:  Nome dell'apertura giocata
  • moves:  Seguenza di mosse che compongono la partita

Ulteriori variabili seppur ignorate dal programma in fase di esecuzione, possono andare a gravare notevolmente sulla memoria.

Si segnala che il programma funziona solamente se la variabile winner è secondo una delle seguenti alternative: 

- "white", "black", "draw"
- "1-0", "0-1", "1/2-1/2"

Si segnala infine che il limite di dimensione del file è impostato a 256 mB, per caricare un file di dimensione maggiore modificare secondo le proprie preferenze la seguente linea del file global.R 

options(shiny.maxRequestSize=256*1024^2)

Alla chiusura il programma re-imposta automaticamente il valore di default di R a 5 mB.
