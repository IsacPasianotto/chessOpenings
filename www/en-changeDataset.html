The program can also be used with its own dataset.

To work correctly, the input file must have a .csv extension. Moreover it must contain the following variables named in this way:

  • winner:   Winner of the match.
  • white_rating:  ELO score of the player with the white pieces.
  • black_rating:  ELO score of the player with the black pieces.
  • opening_code:  ECO code that identifies the played game.
  • opening_name:  Name of the opening played in the match.
  • moves: Game move sequences.

Any other variables present will be ignored.

The program only works if the winner variable is formatted according to one of the following two alterantives:

- "white", "black", "draw"
- "1-0", "0-1", "1/2-1/2"

Finally, it should be noted that the file size limit is set to 256mB, to upload a larger file just modify the following line in the global.R file according to your needs:

options(shiny.maxRequestSize=256*1024^2)

The program automatically sets the default value of R to 5mB when it stops.
